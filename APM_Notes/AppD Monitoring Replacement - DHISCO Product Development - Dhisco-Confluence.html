AppD Monitoring Replacement

Skip to end of metadata
  • Created by Deepak Bhoria, last modified by Atul Sharma on Jun 03, 2020
Go to start of metadata

For monitoring, reporting and alerting, we researched on elk stack and prometheus stack. Both of them are quit good and widely accepted in community

 for monitoring and alerting mechanisms.

ParametersAppdynamicsELK stack

PIG stack

ScalingEasy

Easy

Because of inbuilt sharding and api for rebalancing.

https://stackoverflow.com/questions/35717790/how-to-add-a-new-node-to-my-elasticsearch-cluster

Difficult
No inbuilt scaling, Solutions like Thanos, Cortex help but difficult to setup
Running multiple prometheus: https://sysdig.com/blog/challenges-scale-prometheus/
scaling InfluxDb is hard

Dynamic configuration changesEasyEasy
Urls can be added to heartbeat dynamically for changes
Nodes can be added to metric beat to capture remote node metrics
More nodes can be added to ES cluster to scale accordingly

Easy
Node exporter can be added nodes to expose metrics under /prometheus/metrics url
Promethues config can be dyamically updated to include those nodes to reflect changes in real time

Logs AnalysisEasy
https://docs.appdynamics.com/display/PRO45/Configure+Log+Analytics+Using+Source+Rules		Easy
Logs are indexed at same place as metrics. So indexing and correlation between logs and metrics is easy.

Difficult
Its a metric server only. Need to jump across different platform to establish correlation

SecurityHighHigh
communication b/w ES and Kibana is secured by https

Low
Ok for internal network. But not good if grafana or other services are running in external network. They are planning to do TLS encryption in future.

Ease of useEasyHarder to maintain
Lots of applications need to be configured, run and monitored separately in cluster mode
Heartbeat, MetricBeat, Filebeat, APM, Elastic Search, Logstash, Kibana

Easy to deploy and maintain
Prometheus, Influxdb, grafana can be run on single node easily.

Push Custom metrics through codeYesYes
agent api allows to push custom data as tags

Yes
agent api allows to push custom data as tags

Visualization for custom metricsLow Options
Analytics widget options provide limited visualizations		More options
Kibana lets users create custom visualizations which can be added to dashboard

Limited options
Grafana provides custom visualizations, but those are limited by timeseries queries only

Storage requirementHigh although data is stored in disksHighLow
CPU-RAMLowHighLow
ChargesPaid

Free, Open source

Alerting, Machine learning based solutions are paid

Free, Open source

Currently we are testing ELK, PIG stack on valpocdevsys01a.asp.dhisco.com. Dashboard are running at:

Kibana dashboard: http://valpocdevsys01a.asp.dhisco.com:5601/app/kibana#/dashboards

Grafana dashboard: http://valpocdevsys01a.asp.dhisco.com:3000/
